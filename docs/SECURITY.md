# 安全文档

本文档说明了高德地图去推广 LSPosed 模块的安全考虑和最佳实践。

## 安全概述

本模块是一个 LSPosed 模块，用于拦截高德地图中的推广/广告内容。由于涉及系统级 Hook 和网络请求拦截，需要特别注意安全问题。

## 权限要求

### 必需权限

1. **Root 权限**
   - 安装和启用 LSPosed 模块需要 root 权限
   - 访问应用数据目录需要 root 权限

2. **Xposed/LSPosed 框架**
   - 需要安装 LSPosed 框架
   - 需要启用 Edxposed Manager

### 可选权限

1. **USB 调试**
   - 用于开发和调试
   - 可以通过 `adb` 命令管理模块

2. **网络访问**
   - 模块本身不需要网络权限
   - 高德地图需要网络权限

## 安全风险

### 1. 系统稳定性风险

**风险描述：**
- Hook 可能导致高德地图崩溃
- 不当的 Hook 可能影响系统稳定性

**缓解措施：**
- 只 Hook 必要的方法
- 添加异常处理
- 提供禁用选项

### 2. 数据安全风险

**风险描述：**
- 日志可能包含敏感信息（如 API 请求参数）
- 日志文件可能被其他应用访问

**缓解措施：**
- 日志文件存储在应用私有目录
- 定期清理日志文件
- 不记录敏感信息（如密码、token）

### 3. 隐私风险

**风险描述：**
- 模块可能记录用户的网络请求
- 日志可能包含用户行为信息

**缓解措施：**
- 只记录推广相关 API
- 不记录个人位置信息
- 提供日志清理功能

### 4. 恶意软件风险

**风险描述：**
- 模块可能被恶意修改
- 下载的 APK 可能被篡改

**缓解措施：**
- 从可信来源下载
- 验证 APK 签名
- 使用开源代码

## 代码安全

### 1. 输入验证

```java
// 验证 API 名称
private boolean isValidApiName(String apiName) {
    return apiName != null && !apiName.isEmpty() && apiName.length() < 100;
}

// 验证请求数据
private boolean isValidRequestData(String data) {
    return data != null && data.length() < 10000;
}
```

### 2. 异常处理

```java
try {
    // Hook 逻辑
} catch (Throwable t) {
    Log.e(TAG, "Hook 失败", t);
    // 不要让异常传播，避免崩溃
}
```

### 3. 日志安全

```java
// 不记录敏感信息
private String sanitizeLog(String log) {
    // 移除密码、token 等敏感信息
    return log.replaceAll("password=.*?&", "password=***&")
              .replaceAll("token=.*?&", "token=***&");
}
```

### 4. 文件安全

```java
// 设置日志文件权限
File logFile = new File(logDir, "mtop_log.txt");
logFile.setReadable(false, false);  // 仅所有者可读
logFile.setWritable(false, false);  // 仅所有者可写
```

## 网络安全

### 1. 请求拦截

- 只拦截推广相关 API
- 不修改核心功能请求
- 不记录敏感请求参数

### 2. 响应伪造

- 只伪造推广 API 响应
- 伪造空响应 `{}` 或错误码
- 不修改正常 API 响应

### 3. 流量监控

- 不记录完整请求/响应
- 只记录 API 名称和关键参数
- 定期清理日志

## 数据保护

### 1. 日志文件

- 存储在应用私有目录
- 定期清理（建议每天）
- 不上传到云端

### 2. 配置文件

- 不存储敏感配置
- 使用加密存储（可选）
- 定期备份

### 3. 用户数据

- 不收集用户数据
- 不上传用户信息
- 遵守隐私政策
